# Android XML XPath Validator

This micro-site fits well in MDX because the value is not the standalone HTML shell, it is the interactive behavior plus the written explanation around it. Docusaurus gives you both in one page.

<AndroidXmlXpathValidator />

## Why This Works Better In MDX

- The tool can live next to the explanation instead of as an isolated HTML file.
- You can version the logic with the rest of the docs.
- The page can grow into a guide with examples, caveats, and copy-paste selectors.

## Good Use Cases

- testing selectors for Android automation flows
- checking whether namespaced attributes match as expected
- inspecting matched nodes as either raw XML or a structured tree
- keeping small utility tools inside the docs instead of separate demo files

## Notes

- This runs in the browser using `DOMParser`, `XPathEvaluator`, and `XMLSerializer`.
- It is best suited for well-formed XML snippets and layout dumps.
- XPath support depends on the browser runtime used by the docs page.

## Example XPath Ideas

```xpath
//android.widget.TextView
//android.widget.FrameLayout[@android:clickable='true']
//*[@android:id='@android:id/content']
```
