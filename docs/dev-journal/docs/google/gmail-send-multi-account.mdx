# Gmail Send Multi-Account

The original HTML tool was not just a note. It had a real workflow: configure OAuth, add accounts, compose a message, then send through Gmail's API. This page keeps that workflow interactive inside the docs.

<GmailSendDemo />

## What This Rebuild Preserves

- multiple local accounts
- OAuth URL construction for client-side flow
- RFC 2822 email formatting
- base64url encoding preview
- Gmail API request body shape

## Required Scopes

```text
https://www.googleapis.com/auth/gmail.send
https://www.googleapis.com/auth/userinfo.email
https://www.googleapis.com/auth/userinfo.profile
```

## Production Note

The original micro-site used client-only implicit flow for personal tooling. That is fine for demos and private tools, but a production mail workflow should prefer a backend authorization-code flow.
